# Code of Conduct

## Our Standards

`autonomous-dev-loop` is a public open-source project. Contributors and maintainers are expected to keep collaboration respectful, practical, and focused on improving the project.

Examples of expected behavior:

- Use welcoming and inclusive language.
- Respect different experience levels, workflows, and tools.
- Give and receive technical feedback constructively.
- Keep discussions focused on the project and its users.
- Avoid private, confidential, or company-specific information in public issues or pull requests.

Examples of unacceptable behavior:

- Harassment, insults, threats, or personal attacks.
- Discriminatory language or behavior.
- Publishing private information without permission.
- Deliberately disruptive comments or repeated off-topic behavior.
- Attempts to pressure maintainers into unsafe release, security, or publishing actions.

## Reporting

Report conduct concerns through GitHub issues or repository maintainer channels when available.

If a report involves private information, avoid posting sensitive details publicly. Open a minimal issue asking for a private maintainer contact path instead.

## Enforcement

Maintainers may remove comments, close issues, reject contributions, or restrict participation when behavior is inconsistent with this code of conduct.

Enforcement should be proportionate, transparent when possible, and focused on preserving a safe and useful project environment.
